02 Versioning — one source of truth

appVersion in gradle.properties is the only place the version is set. It drives every consumer, so nothing drifts:

  • desktop .dmg / .msi / .deb packageVersion
  • Android versionName
  • the server artifact version
  • the generated runtime constant AppVersion.NAME (core:models) — used by analytics, the Sentry release tag, and the in-app update check

CI overrides it from the pushed git tag (-PappVersion=1.2.0), so a tagged build always reports the tagged version.

03 One-time setup

Three things must be true before a tag produces a published release.

WhatWhyStatus
Public repo m2f-kt/NoType-releasesThe private source repo can't hand out downloads to un-authenticated clients; this public repo is the download + update channel.done
Secret RELEASE_PAT on the private repoThe release workflow runs in the private repo; its built-in token can't write to a different repo. A fine-grained PAT (resource owner m2f-kt, repo NoType-releases, Contents: read & write) lets CI publish here.todo

Add the PAT secret

UI: private repo → Settings → Secrets and variables → Actions → New secret RELEASE_PAT. Or CLI:

gh secret set RELEASE_PAT --repo m2f-kt/NoType   # paste the PAT when prompted

04 Cut a release

Bump the version, commit, tag, push. The tag is what triggers CI.

# 1. Bump the version (single source of truth)
#    edit gradle.properties → appVersion=1.2.0
git commit -am "chore: release 1.2.0"

# 2. Tag it — this triggers the Release workflow
git tag v1.2.0
git push origin master --tags

You can also run it by hand: Actions → Release → Run workflow, enter a version. No local Xcode/JDK needed — the runner builds it.

05 What CI does

On a v* tag, .github/workflows/release.yml (Apple-Silicon runner):

  1. builds :composeApp:packageReleaseDmg (ProGuard, < 80 MB) with the version from the tag;
  2. renames + SHA-256-checksums the artifact (TypeNot-<version>-arm64.dmg);
  3. publishes a GitHub Release into this public repo with the .dmg and its .sha256 attached, using the RELEASE_PAT.
Verify a download
shasum -a 256 -c TypeNot-1.2.0-arm64.dmg.sha256

06 Signing & notarization

Signing and notarization are env-gated in composeApp/build.gradle.kts — the build auto-enables them when the secrets exist, and ships ad-hoc otherwise.

StateBehavior
today · ad-hocNo signing secrets → ad-hoc-signed, un-notarized .dmg. macOS Gatekeeper blocks silent self-install, so the app notifies + opens the download page.
after Apple enrollmentAdd MACOS_SIGN_IDENTITY + NOTARIZATION_APPLE_ID / NOTARIZATION_PASSWORD / NOTARIZATION_TEAM_ID secrets → CI signs + notarizes. The update can then become one-click download → verify → relaunch.
Before enabling signing

Move the fat-jar native-strip step ahead of signing (see the "Revisit when notarization goes live" comment in composeApp/build.gradle.kts) — stripping after signing invalidates the signature.

07 In-app auto-update

The desktop app checks this repo's latest release once per launch (composeApp/.../update/):

  • polls …/repos/m2f-kt/NoType-releases/releases/latest (public, no auth, bounded timeouts, HTTPS-only, silent on failure);
  • compares the tag to AppVersion.NAME with a small semver comparison;
  • if newer, shows an Aurora prompt — Download opens the release page, Later dismisses; both remember the version so it won't re-nag until an even newer build ships.

Desktop-only; on mobile/web the checker is absent and the prompt is a no-op (those platforms update through their app stores). Fully-silent background install is not possible for direct macOS distribution by Apple's design — one-click prompt-relaunch is the ceiling.

08 Cost & runners

The build runs on a macOS runner in a private repo, the most expensive combination on GitHub Actions:

  • Free tier = 2,000 minutes/month for private repos (public repos are unlimited).
  • macOS bills at 10× → ~200 macOS-minutes on the free allowance.
  • The default spending limit is $0, so once the free minutes are gone, jobs are blocked rather than billed.

Check usage + the spending limit under org billing. Cheapest ways to keep shipping:

OptionTrade-off
Self-hosted macOS runner (your own Mac)$0 GitHub minutes, no quota. Set runs-on: [self-hosted, macOS]. Best for a solo builder. recommended
Build locally, publish manuallyNo CI: ./gradlew :composeApp:packageReleaseDmg -PappVersion=1.2.0 then gh release create v1.2.0 --repo m2f-kt/NoType-releases *.dmg.
Raise the spending limitA few $/month covers a monthly macOS build; keep the tag-triggered workflow as-is.
